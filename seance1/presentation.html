Créer une page WEB

Introduction

Une page WEB est une suite d'instructions notées sur une page de traitement de texte (le fichier source).
Lorsque le navigateur lit cette page, il interprète ces instructions et les affiche sur l'écran.

Exercice: Télécharger le logiciel Notepad++ (si nécessaire) et voir le fichier source de cette page WEB avec ce logiciel.

Balises HTML

Chaque élément d'une page est composé d'une paire de balises HTML. Plus précisément, il faut :

  • Une balise ouvrante de la forme < balise > ;
  • Une balise fermante de la forme < /balise > ;

Exemple: Pour créer un paragraphe, on utilise les balises < p > et < /p >.

  • Identifier les balises utilisées pour créer des listes à puces.

Il existe également des balises dites "orphelines". Ces balises sont seules et elles "s'ouvrent et se referment" directement.
c'est le cas de la balise < br/ > qui permet de revenir à la ligne dans un paragraphe.

D'autres balises sont également très utiles : les balises de commentaire.
Elles se présentent sous cette forme < !-- commentaire -- >. On peut y insérer un commentaire dans le fichier source sans que le navigateur ne l'affiche à l'écran.

Structure d'un document HTML

Un document HTML est composé d'un en-tête dont les balises sont < head > et < /head >.
On y insère par exemple le titre du document et des instructions qui permettent au navigateur de déterminer l'encodage de l'écriture.

Un document HTML est également composé d'un corps dont les balises sont < body > et < /body >.
On y insère l'ensemble des éléments de la page (paragraphes, titres...)

Autrement dit, on intègre directement le style dans la balise ouvrante.
Mais il est vivement conseillé d'utiliser la méthode suivante:

Cet élément est une remarque avec un style CSS

Ici, on décide d'appliquer un style identique à toutes les remarques du document.
Afin d'éviter de recopier à chaque fois la même chose, on crée une "classe" nommée "remarque" avec un style particulier définie dans une page de style CSS.

Vérification du code source

Après avoir créé le fichier source d'une page WEB, il faut vérifier qu'il n'y a pas d'erreur.
Pour cela, on utilise le W3C validator (accessible sur internet) et on teste le code source.
